Canyon 接続ガイド

このガイドでは、Gitサーバー「Canyon」に接続するために必要な設定を順番に案内します。
所要時間は初回のみ10分ほどです。

Canyon とは？

「Canyon」は、モデリングコミュニティ向けに用意された非公開のGitサーバーです。
大容量ファイルの管理（Git LFS対応予定）や、メンバー間のプロジェクト共有が行えます。
サーバーは公開ドメイン portal.dencat.dev 経由でアクセス可能です。外部との通信は暗号化され、安全に保護されています。

Step 1: Gitea アカウントの確認

Gitサーバー（Gitea）へのアクセスには専用のアカウントが必要です。
管理者がアカウントを作成し、個別にユーザー名とパスワードを連絡します。

1. 管理者から連絡されたユーザー名とパスワードを手元に用意してください。
2. Gitea にログインできることを確認してください。

※ HTTPS でクローンする際も、このユーザー名とパスワード（またはトークン）を使用します。

Step 2: TortoiseGit のインストール

Windowsで Git を GUI で操作するためのツール「TortoiseGit」をインストールしてください。
右クリックメニューからclone・push・pullができるようになります。

🐢 TortoiseGit をダウンロード ↗
インストール手順
  1. 上のリンクから TortoiseGit（64-bit）をダウンロードしてインストールする。
  2. インストール後、任意のフォルダを右クリックして「TortoiseGit」メニューが表示されることを確認する。
  3. 初回起動時に「Git for Windows」が必要と案内される場合は、同ページから Git for Windows もインストールする。

Step 3: HTTPS でクローンする

準備ができたら、TortoiseGit を使ってリポジトリをダウンロード（クローン）しましょう！
(Org) と (Repo) は管理者に確認してください。

https://portal.dencat.dev/(Org)/(Repo).git
クローン手順（HTTPS）
  1. クローン先にしたいフォルダを右クリックし、「Git クローン(複製)...」を選択。
  2. 「URL」欄に上のURLを入力する。
  3. 「OK」を押すと、Gitea のユーザー名とパスワードを求められるので入力してください。
  4. クローンが完了したら成功です！
もっと便利に使いたい（SSH設定）

プッシュのたびにパスワードを入力するのが面倒な場合は、SSH 鍵を設定するのがおすすめです。
設定したい場合は、以下のボタンから高度な設定に進んでください。

...
SSH設定ファイルの配置場所

ダウンロードした config ファイルを、以下の場所に配置してください。

  • Windows: C:\Users\あなたのユーザー名\.ssh\config
  • macOS / Linux: ~/.ssh/config

ファイルの中身はこのようになっています：

Host canyon
    HostName portal.dencat.dev
    Port 222
    User git
    IdentityFile ~/.ssh/id_ed25519

Step 5: 公開鍵を Gitea に登録する

前のステップでコピーした公開鍵を、Giteaのアカウント設定に登録してください。
Gitea → SSH鍵の設定ページを開く ↗

登録手順
  1. 上のリンクからGiteaのSSH鍵設定ページを開く。
  2. 「鍵を追加する」または「Add Key」をクリック。
  3. 「タイトル」に任意の名前（例: my-pc）を入力。
  4. 「コンテンツ」欄に、前のステップでコピーした公開鍵を貼り付ける。
  5. 「鍵を追加する」ボタンで保存。

登録後は、URL を ssh:/... にすることで SSH 接続が可能になります。
